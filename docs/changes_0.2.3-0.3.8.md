# 0.2.3 → 0.3.9 更新概要

## 可视化更顺手
可视化页面的入口和操作更“像软件”了：打开 `Visualizer` 会自动切到最近一次运行的输出目录并加载 `*.txt` 文件列表。`Phimax_*.txt` 这类快照会启用 `Plot` 热图显示，现在直接用滚轮就能缩放，左键按住即可拖动查看细节，也支持一键 Fit 和导出 PNG。

## 文本结果更易读
针对常见输出做了“懂你”的解析：`run_config.txt` 会按 `key/value` 两列展示，不再挤成一列；`checkpoint_timestamps.txt` 会补充可读时间、累计耗时和相邻间隔，让你更直观地看每段计算耗时与整体进度。

## 扫参更灵活
Sweep 从“写死几个参数”升级为“每个参数都可选模式”：既可以固定一个值，也可以用开始/结束/步长生成范围，还可以直接输入逗号分隔的数值列表。界面会给出预览与组合数量提示，降低配置门槛，也更不容易配错。

## 运行体验更稳
单次与批量运行加入了 step 级进度条与总进度，让“在跑到哪了”一眼就能看出来。界面布局和样式做了多轮微调，并禁用了前两页参数输入框的滚轮改值，避免滚动页面时误触导致参数被悄悄改掉。

## 打包与细节打磨
应用与发布产物命名统一为 `MID Nano`，更方便识别与分发。针对不同平台的一些小坑（比如目录选择、导出对话框、表格列宽、页面顶部多余边线等）也逐步补齐了处理与日志提示，用起来更顺畅。
